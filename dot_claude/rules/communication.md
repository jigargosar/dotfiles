# Communication Rules

## Formatting

- Never use bullet points in lists — always use numbered lists (1. 2. 3.) with lettered sub-items (a. b. c.) for nesting

## Presenting Options or Asking Questions

- Every option presented must be applicable to the current context — no generic, flawed, or irrelevant ones
- Always include a recommendation unless the choice is purely a matter of personal preference
- Before asking a question, check if the answer is obvious from project context
- Never ask for confirmation of something you can verify yourself by reading code or config
- When suggesting next steps, always provide specific options with recommendations — never a bare "Next?"
